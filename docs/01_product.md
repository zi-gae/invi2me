# 1. 제품 정의

## 1.1 제품 한 줄 정의

**결혼식용 모바일 초대장 제작 도구가 아니라, 초대 → 응답 → 운영 → 체크인 → 리포트까지 연결되는 이벤트 운영 플랫폼**으로 정의한다.

## 1.2 핵심 포지셔닝

- B2C: 커플을 위한 프리미엄 모바일 웨딩 초대/운영 서비스
- B2B Lite: 웨딩홀/플래너/스튜디오가 고객 이벤트를 대신 제작/운영할 수 있는 관리 플랫폼
- B2B Full: 기업 행사, 브랜드 초청 행사, VIP 이벤트, 세미나, 설명회까지 확장 가능한 이벤트 SaaS

## 1.3 벤치마킹 핵심 해석

Splash의 핵심은 단순 랜딩페이지 생성기가 아니라 아래를 하나로 묶는 점이다.

- 브랜드/테마 시스템
- RSVP/티켓/등록
- 이메일 및 링크 기반 유입 운영
- 게스트 데이터 관리
- 행사 당일 체크인
- 실시간 리포팅 및 성과 측정

따라서 본 제품도 아래 원칙을 반드시 유지한다.

1. **페이지와 데이터가 분리되지 않아야 한다.**
2. **디자인 시스템과 운영 시스템이 한 플랫폼 내에 있어야 한다.**
3. **게스트 개별 응답/행동 데이터가 리포트로 연결되어야 한다.**
4. **개인용 웨딩 UX 위에서 B2B 운영 기능이 자연스럽게 확장되어야 한다.**

# 2. 완성형 제품 범위

## 2.1 대상 이벤트 타입

초기 릴리즈 시점부터 이벤트 타입을 고정 문자열이 아니라 확장 가능한 enum/slug 구조로 설계한다.

- wedding
- wedding_reception
- after_party
- baby_shower
- first_birthday
- bridal_shower
- private_party
- corporate_invitation
- seminar
- vip_event

UI 상에서는 첫 출시 시 wedding 중심으로 보이더라도, 데이터 모델은 범용 이벤트 플랫폼으로 설계한다.

## 2.2 릴리즈 스코프

완성형 출시 기준 필수 범위:

### 공개 사용자 기능
- 모바일 이벤트 페이지/청첩장
- 반응형 디자인
- 다크/라이트와 무관한 안정적인 테마 렌더링
- RSVP 응답
- 참석 인원/식사/동반인/추가 질문 응답
- 일정 및 장소 안내
- 지도 및 길찾기
- 갤러리
- 계좌/마음전달/선물 안내
- 연락처/혼주 연락
- FAQ/주차/셔틀/드레스코드/안내사항
- 다중 일정(본식/피로연/2부/애프터)
- 다국어 페이지
- 비공개 접근 제어(비밀번호, 초대코드, 개인화 링크)

### 운영자 기능
- 이벤트 생성/복제
- 템플릿/테마 변경
- 블록 기반 콘텐츠 편집
- 게스트 업로드/세그먼트 관리
- RSVP 실시간 집계
- 체크인 관리
- QR 체크인
- 알림 메시지 발송
- 이메일/SMS/카카오 공유 링크 관리
- 좌석/테이블 관리
- 페이지 성과 리포트
- 유입 채널/공유 링크 성과 추적
- 접근 권한 제어

### B2B 기능
- 조직 단위 워크스페이스
- 다중 이벤트 운영
- 조직 브랜딩
- 템플릿 라이브러리
- 역할 기반 접근 제어
- 운영자/플래너/스태프 계정
- 커스텀 도메인
- 화이트라벨 옵션
- 조직 레벨 리포팅
