# 18. 추가 제안

## 18.1 제품 차별화 기능

### 개인화 초대 링크
- 게스트별로 이름, 동반인 여부, 안내 문구가 다르게 보이도록 지원

### 동행/식사/셔틀 기반 운영 분류
- 일반 RSVP를 넘어서 실제 운영에 필요한 조건을 구조화

### 하객 경험 강화
- 일정 저장(ics)
- 지도 앱 열기
- 언어 전환
- 개별 알림
- 신랑/신부 측별 분기 UX

### 운영 자동화
- RSVP 미응답자 리마인드
- 행사 하루 전 자동 알림
- 체크인 이후 thank-you 메시지 발송

## 18.2 향후 고도화

- AI 카피 생성
- AI 기반 섹션 추천
- 자동 갤러리 레이아웃 추천
- 게스트 군집 분석
- 웨딩홀용 운영 OS
- 기업 이벤트용 Lead capture 모드

# 19. 최종 결론

이 제품은 처음부터 아래 3가지를 동시에 만족하도록 만들어야 한다.

1. **사용자 관점에서는 아름답고 직관적인 모바일 청첩장/이벤트 페이지**
2. **운영자 관점에서는 RSVP, 체크인, 좌석, 메시징, 리포트가 연결된 이벤트 운영 도구**
3. **사업 관점에서는 웨딩 B2C에서 시작해 웨딩 B2B와 기업 이벤트 SaaS로 확장 가능한 플랫폼**

구현 목표는 '청첩장 서비스'가 아니라 **event operating system**이다.
